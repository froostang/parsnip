# Optional calibration

Use calibration only when the user requests it. Save the current task and explain
that this is a rough, session-only pacing estimate—not an ability or reading score.

Present one sample at a time. Ask the user to reply **easy**, **comfortable**,
**dense**, **interrupted**, or **stop** as soon as its meaning feels clear.

1. A blue token opens the garden gate.
2. The morning report goes to the team lead, unless it contains payroll data, in
   which case it goes only to finance.
3. The workshop opens when a member scans a badge. Guests also need a member escort,
   while deliveries use the side entrance before noon; the manager's key overrides
   all three rules during an emergency.

Handle each response:

- **dense:** select the prior level, or level 1 for the first sample, and stop.
- **comfortable:** record that level and offer the next sample.
- **easy:** offer the next sample; if all are easy, select level 3.
- **interrupted:** discard the sample and offer to repeat it or stop.
- **stop:** keep the last comfortable level or level 1.

Use labels rather than message timing. Resume the saved task immediately after the
calibration ends and show the selected density with the controls.
